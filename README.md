# Автоматизированные тесты для сервиса «Яндекс.Самокат»

## Описание проекта

В этом проекте реализованы автотесты для учебного сервиса «Яндекс.Самокат», разработанного специально для студентов. Цель проекта — автоматизация тестирования функциональности сервиса с использованием Java, JUnit и Selenium.

## Технологии

| Технология          | Версия      |
|---------------------|-------------|
| Java                | 11          |
| JUnit               | 4.13.2     |
| Maven               | 3.9.0       |
| Selenium            | 4.20.0      |
| WebDriverManager    | 5.9.3       |

## Настройка проекта

1. Создайте Maven-проект в IntelliJ IDEA.
2. Убедитесь, что у вас установлена Java 11.
3. Подключите зависимости JUnit и Selenium в вашем `pom.xml`:

   ```xml
   <dependencies>
       <dependency>
           <groupId>junit</groupId>
           <artifactId>junit</artifactId>
           <version>4.13.2</version>
           <scope>test</scope>
       </dependency>
       <dependency>
           <groupId>org.seleniumhq.selenium</groupId>
           <artifactId>selenium-java</artifactId>
           <version>4.20.0</version>
       </dependency>
       <dependency>
           <groupId>io.github.bonigarcia</groupId>
           <artifactId>webdrivermanager</artifactId>
           <version>5.9.3</version>
       </dependency>
   </dependencies>
4. Установите Google Chrome и Mozilla Firefox для выполнения тестов.

## Тестовые сценарии

1. Выпадающий список:
   Проверить, что при нажатии на стрелочку открывается соответствующий текст.
2. Заказ самоката:
- Нажать кнопку «Заказать» (две кнопки вверху и внизу страницы).
- Заполнить форму заказа.
- Проверить появление всплывающего окна с сообщением об успешном создании заказа.
- Использовать минимум два набора данных для тестирования.

## Запуск тестов

Чтобы запустить тесты, выполните следующие шаги:
1. Убедитесь, что у вас установлены все зависимости и драйверы браузеров.
2. Запустите тесты через Maven командой: mvn clean test

## Примечание

Обратите внимание, что в приложении есть баг, который может помешать оформлению заказа. Это может привести к падению тестов, что полезно для обнаружения неисправностей.
Используйте Git для управления версиями и хранения кода.
